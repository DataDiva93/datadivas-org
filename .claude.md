# Data Divas Project Context

## Overview
Data Divas is a community platform for data engineers, data scientists, and ML engineers. We're all about shipping real systems — labs, repos, pipelines, demos, and portfolios.

**Tagline:** "The data community that ships."

## Brand Identity

### Color Palette
Purple-based tech aesthetic with the following color scheme:

- **Primary Purple:** `#8b5cf6` (--purple-500)
- **Dark Purple:** `#5b21b6` (--purple-700)
- **Light Purple:** `#c4b5fd` (--purple-300)
- **Accent Colors:**
  - Cyan: `#06b6d4` (for links, terminal prompts)
  - Green: `#10b981` (for success, terminal $)
  - Pink: `#ec4899` (for gradients)
  - Orange: `#f97316` (for highlights)

### Dark Theme
- Background: `#0f0a1f` (--bg-dark)
- Card Background: `#1a1425` (--bg-card)
- Border: `#2d1b4e` (--border-default)
- Text Primary: `#f3f4f6`
- Text Secondary: `#9ca3af`

## Typography

### Font Stack
- **Headings:** JetBrains Mono, monospace, bold (700)
- **Body Text:** Inter, sans-serif
- **Code/Metrics:** JetBrains Mono, monospace

### Typography Guidelines
- H1: JetBrains Mono, gradient purple-to-pink effect
- H2: JetBrains Mono, purple-300 color, prefixed with `//`
- Code blocks: Terminal-style with `$` prompts
- Buttons: Monospace font with function-style text like `join_discord()`

## Tech Stack

### Frontend
- **Framework:** Astro (v5+)
- **Documentation:** Starlight (Astro integration)
- **Deployment:** Cloudflare (via @astrojs/cloudflare adapter)
- **Styling:** Pure CSS (no Tailwind, no CSS-in-JS)

### Key Dependencies
- `astro` - Static site framework
- `@astrojs/starlight` - Documentation theme
- `@astrojs/cloudflare` - Cloudflare adapter
- `@astrojs/sitemap` - Sitemap generation

## Design System

### CSS Variables Location
All design tokens are defined in `/src/styles/global.css` under the `:root` selector.

### Reusable Classes
- `.card` - Basic card component
- `.lab-card` - Lab-specific card with hover effects
- `.badge` - Tech badges with variants (.badge-green, .badge-cyan, etc.)
- `.tech-badge` - Technology stack badges
- `.btn`, `.btn-primary`, `.btn-outline`, `.btn-terminal` - Button styles
- `.terminal` - Terminal-style code blocks
- `.stat`, `.stat-label` - Metrics/statistics display
- `.container`, `.container-narrow` - Layout containers

### Design Principles
1. **Nerdy/Geeky Tech Vibe** - Monospace fonts, terminal aesthetics, code-like elements
2. **Function Syntax** - CTAs styled like function calls: `join_discord()`
3. **Code Comments** - Use `//` prefix for section headers
4. **Terminal Prompts** - Use `$` or `>` for commands and CTAs
5. **Syntax Highlighting Colors** - Use accent colors for different tech stacks
6. **Glow Effects** - Subtle purple glows on hover for interactive elements
7. **Dark Mode First** - Designed for dark backgrounds with bright accent colors

## Content Guidelines

### Tone
- Direct, no-nonsense
- Focus on shipping and building
- Celebrate real accomplishments
- Use tech/dev lingo naturally
- Avoid corporate speak

### Writing Style
- Keep it concise
- Use code metaphors
- Reference real tools (Docker, Postgres, Airflow, dbt)
- Show, don't just tell (link to actual repos and demos)

5. Prefer dark backgrounds with bright accent colors
6. Add terminal-style elements where appropriate
7. Use badges to highlight tech stacks
8. Include hover effects with subtle purple glows
